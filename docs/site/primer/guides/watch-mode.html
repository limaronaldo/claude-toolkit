Watch Mode

Monitor your project for changes and regenerate documentation automatically.

Basic Usage

# Watch with manual confirmation on each change
claude-primer --watch

# Auto-regenerate without prompts
claude-primer --watch --watch-auto

# Custom polling interval (default: 5 seconds)
claude-primer --watch --watch-interval 10

How It Works

  1. Performs an initial scan and generation
  2. Monitors key project files (package.json, requirements.txt, etc.) for modifications
  3. When changes are detected, reports what changed
  4. Regenerates documentation (with confirmation unless --watch-auto is set)

Watched Files

Watch mode monitors files that affect project detection:

  • Package manifests: package.json, requirements.txt, pyproject.toml, Cargo.toml, go.mod
  • Config files: Dockerfile, docker-compose.yml, Makefile
  • CI configs: .github/workflows/*.yml

Combining with Other Flags

# Watch with template overrides
claude-primer --watch --watch-auto --template-dir ./my-templates

# Watch and also generate agent context
claude-primer --watch --watch-auto --agent all

Watch mode always runs with --force and --git-mode skip to avoid interactive prompts during regeneration.
