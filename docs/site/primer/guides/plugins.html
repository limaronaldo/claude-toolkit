Plugin System

Register custom generators to produce additional files alongside built-in output.

Setup

mkdir -p .claude-primer/plugins

Writing a Plugin (Python)

Create a .py file with a generate(info) function: 

# .claude-primer/plugins/api-docs.py

def generate(info):
    """Generate API documentation scaffold."""
    stacks = ", ".join(info.get("stacks", []))
    frameworks = ", ".join(info.get("frameworks", []))

    return {
        "filename": "API_DOCS.md",
        "content": f"""# API Documentation

## Tech Stack
{stacks}

## Frameworks
{frameworks}

## Endpoints


"""
    }

Writing a Plugin (Node.js)

Create a .mjs file with a default export function: 

// .claude-primer/plugins/api-docs.mjs

export default function generate(info) {
  return {
    filename: "API_DOCS.md",
    content: `# API Documentation\n\nStack: ${(info.stacks || []).join(", ")}\n`
  };
}

Multi-File Plugins

Return an array to generate multiple files:

def generate(info):
    return [
        {"filename": "docs/API.md", "content": "# API\n"},
        {"filename": "docs/ARCHITECTURE.md", "content": "# Architecture\n"},
    ]

Plugin Contract

The info dict contains the full project scan result:

  • info["name"] — project directory name
  • info["stacks"] — detected language stacks (list)
  • info["frameworks"] — detected frameworks (list)
  • info["tier"] — tier classification dict
  • info["deploy"] — deploy targets (list)
  • info["is_monorepo"] — boolean
  • info["existing_content"] — extracted docs content

Error Handling

If a plugin throws an exception, claude-primer prints a warning and continues with the remaining plugins and built-in generation. Your plugin will not crash the CLI.

Custom Plugin Directory

claude-primer --plugin-dir path/to/my-plugins
