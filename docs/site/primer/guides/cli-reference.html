CLI Reference

Synopsis

claude-primer [target] [options]

Positional Arguments

Argument Description
target Target directory (default: current directory)

Core Options

Flag Description
--dry-run Preview changes without writing files
--force Overwrite existing files (skip unchanged)
--force-all Overwrite all files unconditionally
--yes, -y Accept defaults, no interactive prompts
--plan-json Output project analysis as JSON without writing
--check Check if generated docs are up-to-date (exit 1 if stale)
--export [FILE] Export docs to a file (default: claude-primer-export.md, combined markdown). Specify out.tar.gz or out.zip to produce an archive instead.
--reconfigure Re-run wizard (ignores saved .claude-setup.rc)

Content Options

Flag Description
--with-readme Include README.md in generated files
--with-ralph Generate Ralph integration files
--from-doc <file|URL> Bootstrap from an existing document (PRD, spec, etc.). Accepts local file paths and HTTP/HTTPS URLs.
--clean-root Move auxiliary docs to .claude/docs/

Git Safety

Flag Description
--no-git-check Skip git safety entirely
--git-mode ask|stash|skip Git safety mode (default: ask)

Template System

Flag Description
--template-dir <dir> Template overrides directory (default: .claude-primer/templates/)

Watch Mode

Flag Description
--watch Watch for changes and suggest updates (poll-based)
--watch-interval <secs> Polling interval in seconds (default: 5)
--watch-auto Auto-regenerate without confirmation

Multi-Agent Output

Flag Description
--agent <agents> Target agents: claude, cursor, copilot, windsurf, aider, codex, all (comma-separated)
--format json|yaml|markdown Output format for generated files (default: markdown). JSON and YAML produce structured knowledge files.

Plugin System

Flag Description
--plugin-dir <dir> Plugin generators directory (default: .claude-primer/plugins/)

Diff and Config

Flag Description
--diff Show what would change without writing (unified diff). Fixed in v1.8.1: no longer loops infinitely when comparing an empty file to new content (npm).

See also: Config File for .claude-primer.toml usage.

Telemetry

Flag / Env Var Description
--telemetry-off Disable telemetry even if env var is set
CLAUDE_PRIMER_TELEMETRY=1 Opt-in to anonymous usage analytics. When opted in, all standalone commands (--check, --export, --migrate, --init, --update, --diff) emit telemetry events.

Examples

# Full automation: no prompts, overwrite all, skip git
claude-primer /path/to/repo --force-all --yes --no-git-check

# Generate for all AI tools
claude-primer --agent all --yes

# Watch with auto-regeneration and custom templates
claude-primer --watch --watch-auto --template-dir ./templates

# Show what changed since last generation
claude-primer --diff

# CI/CD pipeline usage
claude-primer /workspace --yes --no-git-check --plan-json > analysis.json

Structured Output

# JSON knowledge file
claude-primer --format json --yes

# YAML knowledge file
claude-primer --format yaml --yes

# Check freshness in CI
claude-primer --check --no-git-check

# Export as combined markdown (default)
claude-primer --export

# Export as zip or tar.gz archive
claude-primer --export docs-bundle.zip
claude-primer --export docs-bundle.tar.gz
