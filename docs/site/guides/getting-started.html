Getting Started

Installation

Choose the method that works best for your environment:

Zero-install (recommended)

# Python (pipx)
pipx run claude-primer

# Python (uvx)
uvx claude-primer

# Node.js
npx claude-primer

Standalone binary

# Linux / macOS
curl -fsSL https://raw.githubusercontent.com/limaronaldo/claude-primer/main/install.sh | bash

# Windows (PowerShell)
irm https://raw.githubusercontent.com/limaronaldo/claude-primer/main/install.ps1 | iex

Package managers

# macOS
brew install limaronaldo/tap/claude-primer

# Windows (Scoop)
scoop bucket add limaronaldo https://github.com/limaronaldo/scoop-bucket
scoop install claude-primer

# pip (global install)
pip install claude-primer

Docker

docker run --rm -v "$(pwd):/project" ghcr.io/limaronaldo/claude-primer

Basic Usage

# Interactive setup in current directory
claude-primer

# Specific directory, non-interactive
claude-primer /path/to/project --yes

# Preview without writing
claude-primer --dry-run

# Overwrite existing files (skip unchanged)
claude-primer --force --yes

# Output project analysis as JSON
claude-primer --plan-json

# Structured output (JSON or YAML)
claude-primer --format json --yes
claude-primer --format yaml --yes

# Check if docs are up-to-date (great for CI)
claude-primer --check --no-git-check

What Happens

  1. Scan — Detects languages, frameworks, deploy targets, and monorepo structure
  2. Git safety — Checks for uncommitted changes (configurable via --git-mode)
  3. Generate — Creates CLAUDE.md, STANDARDS.md, QUICKSTART.md, ERRORS_AND_LESSONS.md
  4. Verify — Confirms all files were written correctly
