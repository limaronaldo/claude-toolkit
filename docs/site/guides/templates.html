Template System

Override any section of generated output with your own content. Templates survive --force regeneration.

Setup

Create a .claude-primer/templates/ directory in your project root:

mkdir -p .claude-primer/templates

Template Files

Template filenames map to output files:

Template FileOverrides
claude.mdCLAUDE.md
standards.mdSTANDARDS.md
quickstart.mdQUICKSTART.md
errors.mdERRORS_AND_LESSONS.md

Section-Level Overrides

Use ## Header to override specific sections. Only sections with matching headers are replaced:

# .claude-primer/templates/claude.md

## Code Architecture

Our service follows a hexagonal architecture pattern:
- Domain layer: pure business logic, no framework dependencies
- Ports: interfaces for external interactions
- Adapters: implementations (REST, DB, messaging)

## Formatting Standards

- Use Prettier with 2-space indentation
- All files must pass ESLint before commit

Variable Substitution

Use {{variable}} placeholders:

VariableValue
{{project_name}}Directory name
{{tech_stack}}Detected stacks (comma-separated)
{{frameworks}}Detected frameworks
{{tier}}Tier level (T1-T4)
{{deploy}}Deploy targets
{{date}}Current date (YYYY-MM-DD)
{{description}}Project description

Example

# .claude-primer/templates/claude.md

## Repository Overview

{{project_name}} is built with {{tech_stack}} ({{frameworks}}).
Classified as {{tier}} — last updated {{date}}.

Full File Override

If your template has no ## Header sections, the entire generated file is replaced with your template content.

Custom Template Directory

claude-primer --template-dir path/to/my-templates
