Config File

Persist CLI flag defaults in a .claude-primer.toml file at your project root.

Setup

Create a .claude-primer.toml file in your project root:

# .claude-primer.toml

[flags]
force = true
with_readme = true
clean_root = true
agent = "all"
format = "markdown"

Precedence

CLI flags always override TOML values. TOML values only apply when the CLI flag is at its default.

# TOML says force = true, but CLI says --dry-run
# Result: dry-run mode (CLI wins), force is also set from TOML
claude-primer --dry-run

Diff Mode

Use --diff to preview what would change without writing any files:

# Show unified diff of changes
claude-primer --diff

# Combine with other flags
claude-primer --diff --with-readme --clean-root

Output is in unified diff format, showing additions and removals per file.

Migration from .claude-setup.rc

The .claude-setup.rc file (INI format) is still supported for wizard answers. The .claude-primer.toml file is for CLI flag defaults only. Both can coexist.
