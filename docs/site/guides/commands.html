Commands

--migrate

Convert .claude-setup.rc (INI format) to .claude-primer.toml (TOML format). 

claude-primer --migrate
claude-primer /path/to/project --migrate

Reads the existing .claude-setup.rc, extracts project metadata (description, stacks, frameworks, deploy targets, monorepo settings), and writes a .claude-primer.toml with a [project] section and a [flags] section with commented-out defaults.

The original .claude-setup.rc is NOT deleted — both files can coexist.

--init

Interactively create a .claude-primer.toml config file. 

claude-primer --init
claude-primer --init --yes  # non-interactive with defaults

In interactive mode, asks 5 questions: force mode, include README, clean root, git mode, target agents. In --yes mode, writes sensible defaults (all false, git_mode="ask").

--update

Self-update the compiled binary from GitHub releases. 

claude-primer --update

Only works for compiled binaries (PyInstaller builds). When running from source, suggests pip install --upgrade claude-primer or npm install -g claude-primer@latest instead.

Downloads the platform-appropriate binary, verifies SHA256 checksum, backs up the current binary, and replaces it. Restores from backup on failure.

--check

Validate whether generated docs are up-to-date. Returns exit code 0 if all files match, exit code 1 if any are stale or missing. Ideal for CI pipelines. 

# Check if docs are current
claude-primer --check

# Use in CI (fails the build if docs are stale)
claude-primer --check --no-git-check

Output shows status per file:

  OK      CLAUDE.md
  OK      STANDARDS.md
  STALE   QUICKSTART.md
  MISSING ERRORS_AND_LESSONS.md
  2 file(s) stale or missing.

--export

Bundle all generated docs into a single archive. Supports .tar.gz, .zip, or combined markdown output. 

# Default: tar.gz archive
claude-primer --export

# Custom filename determines format
claude-primer --export docs-bundle.zip
claude-primer --export knowledge.md

# Include agent files if they exist
claude-primer --export --agent all

The archive includes CLAUDE.md, STANDARDS.md, QUICKSTART.md, ERRORS_AND_LESSONS.md, and any agent-specific context files.

--from-doc (URL Support)

Bootstrap from a remote document. Accepts both local file paths and HTTP/HTTPS URLs. 

# From a local file
claude-primer --from-doc ./architecture.md

# From a URL
claude-primer --from-doc https://example.com/spec.md

# From a GitHub raw URL
claude-primer --from-doc https://raw.githubusercontent.com/org/repo/main/ARCHITECTURE.md

URLs are fetched with a 15-second timeout. The content is parsed and used to seed the generated documentation.
